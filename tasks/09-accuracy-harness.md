## 3. scripts/31_bench_accuracy.py
Args: `--base-url --api-key-file --out --stack-label --extra-body --suite gsm8k|mmlu-pro|humaneval --split dev|holdout|all --transcripts-dir` plus `--completions-endpoint /v1/completions` (default) and `--config-hash STR` (identifier of the serving config under test, required for holdout).
- **Splits** (deterministic, no dataset content in code): GSM8K — shuffle indices 0..1318 with random.Random(42) → dev = first 100, holdout = next 100. - **Rendering**: official encoder, non-thinking. GSM8K prompt: question + "\n\nThink briefly if needed, then end with the final numeric answer on its own line in the form: Answer: <number>". MMLU-Pro: question + lettered options + "\n\nReply with the single letter of the correct option in the form: Answer: <letter>". - **Scoring**: gsm8k — extract the LAST `Answer:\s*(-?[\d,\.]+)` (fallback: last number in text); compare numerically (strip commas) to the reference after `#### `. mmlu-pro — extract last `Answer:\s*([A-J])` (fallback: last standalone capital letter A–J); compare. Invalid/timeout/unparseable = incorrect (recorded with reason). humaneval — write completion to a temp dir, run the check inside docker: image `python:3.12-slim` (orchestrator pre-pulls), `docker run --rm --network none --cap-drop ALL --security-opt no-new-privileges --memory 512m --memory-swap 512m --pids-limit 128 --cpus 1 --read-only --tmpfs /tmp -u 65534:65534 -v <case_dir>:/case:ro python:3.12 timeout... ` — construct: /case/main.py = prompt + completion + "\n\n" + test + f"\n\ncheck({entry_point})"; command `timeout 10 python /case/main.py`; exit 0 = pass.
